Qs: How does JS works

Ans:

  • first code is received by parser & check the syntactic errors line by line
  • If there's no errors exists it creates the data structure called AST(Abstract Syntax Tree)
  • Then the JS engine converts the code into machine code
  • After running it on machine code it is sent for execution
  • Finnally JS program run at web browser

Qs: What does JS event loop do?

Ans:

javascript is a single threaded language but browser API's are multi-threaded.The event loop executes this constantly whether it empty or not. If the processing is empty, new function is added from event queue loop. Rather it another function is processed.

const second = ()=>console.log("I'm Second")
const third = ()=>console.log("I'm third")
const first = ()=>{
console.log("I'm first")
setTimeOut(second,2000)
third()
}
first()

এই উপরের কোডটি যেখানে রান হয় সেটাকে বলে call stack.

তিনটা ফাংশনের মধ্যে প্রথমে যেহেতু first() কে কল করা হয়েছে তাই, এটি সবার শুরুতে console এ আউটপুট দিবে।
তারপর second() এবং তারপর third() console এ দিবার কথা। কিন্তু যেহেতু setTimeOut একটি asynchronous function এবং জাভাস্ক্রিপ্ট synchronous and asynchronous দুই ওয়েতেই কাজ করে, তাই second function কে হোল্ড এ রেখে third function console এ আউটপুট হিসেবে দিবে। second function কে web api নামক একটি ইঞ্জিন হোল্ড করে callback queue তে জমা করে রাখে।

এই call stack & callback queue এর মধ্যে যে যোগাযোগ মেইন্টেইন করবে তার নাম Event Loop.
